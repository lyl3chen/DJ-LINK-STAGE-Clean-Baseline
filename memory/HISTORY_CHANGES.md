## 四、经验教训

1. **大模型难以在复杂代码基础上可靠收敛** - 需要干净基线
2. **简化载波无法被专业设备识别** - 必须实现标准 LTC 编码
3. **实时音频链路可能引入抖动** - 需解耦独立线程
4. **停止态边界处理复杂** - 需要明确策略并锁定

## 五、当前保留的设计决策

| 决策 | 说明 |
|------|------|
| 标准 LTC 编码 | 80-bit LTC frame + 标准 BMC 调制 |
| LTC/MTC 共用 TimecodeCore | 统一时间源 |
| sourcePlayer 统一 | sync.masterPlayer 替代 timecode.sourcePlayer |
| 手动测试模式 | 不依赖 DJ Link 独立测试 |
| 独立音频写线程 | 避免实时链路阻塞 |
