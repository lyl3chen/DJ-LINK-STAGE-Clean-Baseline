## 一、当前代码状态梳理

### A. 本地播放器子系统

#### ✅ 已完成

| 功能 | 实现位置 | 状态 |
|------|----------|------|
| 基础播放引擎 | `BasicLocalPlaybackEngine.java` | 可用 |
| Play/Pause/Stop | `PlaybackEngine.java` 接口 | 完整 |
| Seek（跳转） | `BasicLocalPlaybackEngine.seek()` | 完整，支持 PLAYING/PAUSED/STOPPED 三种状态下的跳转语义 |
| 音频设备选择 | `setAudioDevice()` / `getAudioDevice()` | 可用 |
| 格式协商 | `findSupportedFormat()` | 支持 48000/44100、16bit、mono/stereo |
| 资源清理 | `forceCleanup()` / `cleanupAfterFailure()` | 较完善，有 shutdown hook |
| WebUI 控制面板 | `index.html` 本地播放器 tab | 可用 |
| 曲目导入 | 文件上传 + 路径导入 | 可用 |
| 曲目列表 | `/api/local/tracks` | 可用 |
| 播放状态 API | `/api/local/status` | 可用 |
| 删除曲目 | `/api/local/delete` | 可用，PLAYING/PAUSED 状态禁止删除 |

#### ❌ 还缺少

| 功能 | 重要程度 | 说明 |
|------|----------|------|
| **当前输出设备清晰显示** | P1 | 当前只显示 configuredDevice 和 actualOpenedDevice，但界面不够直观 |
| **播放进度条可视化** | P1 | 只有数字显示，没有图形进度条 |
| **切歌提示/确认** | P2 | 播放中加载新曲目时，没有明确提示 |
| **下一首/上一首** | P2 | 列表导航功能 |
| **音量控制** | P2 | 缺少音量调节 |
| **播放模式** | P3 | 单曲循环、列表循环、随机 |
| **曲目搜索/筛选** | P3 | 列表多了之后难以找到 |
| **播放列表持久化** | P3 | 当前播放队列没有保存 |

### B. 曲目分析子系统

#### ✅ 已完成

| 功能 | 实现位置 | 状态 |
|------|----------|------|
| 基础分析器 | `BasicAudioAnalyzer.java` | 可用 |
| BPM 检测 | `estimateBpm()` - onset + 自相关 | 基础算法，60-180 BPM 范围 |
| 置信度计算 | 基于 best/second peak 比值 | 简单实现 |
| WaveformPreview 生成 | `generateWaveformPreview()` | 1000 个峰值点，包络提取 |
| BeatGrid 生成 | `generateBeatGrid()` | 线性网格，假设 firstBeatMs=0 |
| 分析服务 | `AnalysisService.java` | 同步/异步分析，状态流转 |
| 结果模型 | `AnalysisResult.java` | 完整 DTO |
| BeatGrid 模型 | `BeatGrid.java` | 含时间换算工具 |
| WebUI 分析测试区 | `index.html` | 可用，显示 BPM/置信度/waveform/beats |
| 波形预览 Canvas | `drawAnalysisWaveform()` | 基础显示 |

#### ⚠️ 已实现但质量待提升

| 功能 | 当前状态 | 问题 |
|------|----------|------|
| **BPM 检测精度** | 基础算法 | onset + 自相关，对复杂节拍可能不准；没有多候选 BPM |
| **BeatGrid 精度** | 线性推算 | 假设第一拍在 0ms，实际可能有偏移；没有 downbeat 检测 |
| **Waveform 精度** | 1000 点 | 可能不够精细，缺少多分辨率 |
| **置信度语义** | 简化计算 | best/second ratio，可能不够可靠 |

#### ❌ 还缺少

| 功能 | 重要程度 | 说明 |
|------|----------|------|
| **分析结果持久化** | P1 | 目前是 InMemoryRepository，重启丢失 |
| **重复分析策略** | P1 | 没有"是否覆盖"逻辑 |
| **分析进度指示** | P1 | 大文件分析时，WebUI 没有进度条 |
| **分析错误详细提示** | P1 | 错误信息不够具体 |
| **BeatGrid 手动校准** | P2 | 无法调整 first beat 位置 |
| **Waveform 多分辨率** | P2 | 只有固定 1000 点 |
| **分析结果导出/导入** | P3 | 无法备份分析数据 |
| **批量分析** | P3 | 一次分析多首曲目 |

## 二、开发拆分方案

### Phase P1: 基础完善（核心可用性）

**目标**: 让本地播放器和曲目分析达到"可依赖"的基础状态

**修改范围**:

1. **本地播放器**
   - 播放进度条可视化（WebUI 添加进度条组件）
   - 当前输出设备更清晰的显示（优化设备诊断 UI）
   - 切歌时的状态提示（加载新曲目时显示确认或提示）

2. **曲目分析**
   - 分析结果持久化（添加 JSON 文件存储，替代纯内存）
   - 重复分析策略（分析前检查已有结果，提示是否覆盖）
   - 分析进度指示（大文件分析时显示进度）
   - 分析错误详细提示（改进错误信息展示）

3. **曲目库管理**
   - 曲目库持久化（与 analysis 一起存储到文件）

**验证方式**:
- WebUI 可以清晰看到播放进度条
- 重启应用后，已导入曲目和分析结果依然存在
- 重新分析时提示"是否覆盖已有结果"
- 分析大文件时（>30秒）有进度指示
- 分析失败时显示具体错误（如"不支持的格式"、"文件读取失败"等）

**影响现有功能**: 低（主要是添加功能，不改变现有接口）

---

### Phase P2: 体验优化（易用性提升）

**目标**: 让日常操作更流畅

**修改范围**:

1. **本地播放器**
   - 下一首/上一首功能（列表导航按钮）
   - 音量控制（滑块控制 + 静音按钮）
   - 播放模式切换（单曲循环、列表循环、随机）
   - 曲目搜索/筛选（按标题搜索）

2. **曲目分析**
   - BeatGrid 手动校准 UI（调整 first beat 偏移）
   - Waveform 点击定位（点击波形跳转到对应位置）
   - 分析结果对比（重新分析后与旧结果对比）

3. **WebUI 整体**
   - 曲目列表显示更多信息（时长、是否已分析）
   - 已分析曲目标识（列表中高亮或图标）

**验证方式**:
- 可以按"下一首"切换曲目
- 可以调节音量
- 可以搜索曲目
- 可以拖动调整 beat grid 的 first beat 位置

**影响现有功能**: 中（添加新 API 端点，修改 WebUI 布局）

---

### Phase P3: 稳定性增强（生产可用）

**目标**: 达到可以现场使用的稳定程度

**修改范围**:

1. **本地播放器**
   - 播放列表持久化（保存当前播放队列）
   - 异常恢复（播放中断后自动恢复）
   - 音频设备热插拔处理

2. **曲目分析**
   - 分析结果导出/导入（备份和迁移）
   - 批量分析功能
   - BPM 检测算法改进（多候选、更鲁棒）

3. **系统级**
   - 配置文件持久化
   - 日志记录改进

**验证方式**:
- 重启后恢复上次播放列表和位置
- 可以导出/导入分析结果
- 批量分析多首曲目

**影响现有功能**: 中（涉及存储格式变更）

## 三、主推荐实施顺序

### 推荐顺序: **P1 → P2 → P3**

理由:
1. **P1 是基础** - 没有持久化，一切功能都是临时的
2. **P2 是体验** - 基础可用后，再优化操作流程
3. **P3 是 polish** - 最后处理高级功能和边缘情况

### 具体实施建议

**第一步（本周）**: 完成 P1 的核心内容
- 分析结果持久化（JSON 文件存储）
- 播放进度条
- 分析进度指示

**第二步（下周）**: 完成 P1 的剩余内容和 P2 的部分内容
- 曲目库持久化
- 重复分析策略
- 下一首/上一首
- 音量控制

**第三步（再下周）**: 完成 P2 和 P3
- BeatGrid 校准
- 曲目搜索
- 播放列表持久化

## 四、本阶段明确不做的事

根据用户要求，以下功能在本阶段（本地播放器 + 曲目分析完善阶段）**不做**：

- ❌ Trigger 自动触发系统
- ❌ Trigger 配置 UI
- ❌ MA2 / Titan 自动规则
- ❌ AI 分析
- ❌ Marker 编辑（P2 可能涉及简单的 BeatGrid 校准，但不涉及完整 Marker 系统）
- ❌ 波形编辑
