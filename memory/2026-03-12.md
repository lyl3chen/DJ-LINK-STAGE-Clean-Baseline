## 23:33 Ableton Link 当前实现架构说明
- 用户询问当前 Ableton Link 的实现逻辑/架构。
- 已说明现状为单路架构：
  - `SyncOutputManager(统一source派生)` -> `AbletonLinkDriver` -> `CarabinerLinkEngine(lib-carabiner)` -> `Carabiner` -> `Ableton Link`

  3) Engine 周期发送 `status/version/bpm/beat-at-time`，并重发 `enable-start-stop-sync`
  4) 具备断线自动重连与 peer 变化后的会话恢复逻辑
  5) WebUI 语义：Enabled 即启停，不再有旧 bridge Start/Stop
- 总结：Java + lib-carabiner 单实现，已实现 tempo+beat 双同步，并与 LTC/MTC 共用 sourceMode 派生层。

## 23:36 lib-carabiner vs 自研 C++ bridge 对比结论
- 用户询问两者区别、优劣。
- 已给出简版结论：
  - lib-carabiner 优势：Java 集成顺滑、维护成本低、跨平台交付简单
  - lib-carabiner 劣势：底层可控性受限，遇 discovery 边角问题可调空间较小
  - C++ bridge 优势：底层可控性高、可做细粒度现场定制

- 当前项目建议：以稳定与维护优先，继续使用 lib-carabiner；仅在遇到其无法解决的硬问题时再考虑回退 C++ 定制桥。
