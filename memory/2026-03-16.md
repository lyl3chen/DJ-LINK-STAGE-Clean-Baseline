# 2026-03-16 进展记录（LTC/MTC 重构完成）

## 今日主要工作

### 1. LTC/MTC 模块重建完成

**编码器核心重写**（`ddabfbe`）
- 重写 `LtcFrameEncoder`：按 x42/libltc 线序，BCD LSB-first，sync word 0x3FFD
- 重写 `LtcBmcEncoder`：连续相位，严格 BMC 调制规则
- 验证：与标准样本对照 3000 帧，mismatch=0

**架构重构**
- TimecodeCore 完全独立于 SyncOutputManager
- 两者 sourcePlayer 配置分离
- API 字段区分：`sourcePlayer` vs `tcSourcePlayer`

**停止态修复**（`f8ed84d`）
- 停止后 LTC 发送静音帧（全0样本）
- 解决接收端在 -0/0 之间抖动的问题

### 2. 功能补齐

**手动测试模式**
- TimecodeCore 支持 `manualTestMode`
- 脱离 CDJ 独立运行，从 00:00:00:00 线性推进
- WebUI 添加「手动测试」按钮（Timecode Source 卡片内）

**设备枚举**（`497f3e7`）
- AudioDeviceEnumerator：检测物理/虚拟音频设备
- MidiDeviceEnumerator：检测 MIDI 输入/输出端口
- WebUI 自动加载可用设备列表

**实时诊断**（`34088ed`）
- writeInterval：写入间隔统计
- bufferOccupancy：音频缓冲占用
- underrunCount：缓冲饥饿计数
- frameDelta：帧推进差异分析

### 3. 关键修复

**Player state null 处理**（`3d828fc`）
- CDJ `state=null` 时根据 `playing` 标志正确识别状态
- 修复停止后无法恢复播放的问题

**MTC MIDI 设备匹配**（`fb291fd`）
- 修复 `openMidiDevice()` 按名称查找设备的逻辑
- MTC 现在能正确打开指定 MIDI 端口

**WebUI JS 错误**（`54b023b`）
- 移除未定义的 `tcSourceSelected` 变量
- 修复 Timecode Core 状态无法更新的问题

### 4. API 变更

```json
// 修改前
{
  "sourcePlayer": 1,
  "timecode": {
    "sourcePlayer": 2  // 同名冲突
  }
}

{
```
