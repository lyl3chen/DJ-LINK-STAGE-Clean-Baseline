# Incident Closure - Local Player 设备打开失败 + 前端 JSON 解析错误（2026-03-19）

## 状态
- **结论**：已收口，可关闭。
- **现场验证**：歌曲已可正常播放（用户现场实测确认）。

---

## 1) 原始现象
页面先后出现两类报错：

1. 底层错误：
   - `Load error: LineUnavailableException - line with format PCM_SIGNED 48000Hz 16bit stereo ... not supported`
2. 前端错误：
   - `Unexpected token '<' ... is not valid JSON`

---

## 2) 最终归因（统一问题链）
这不是“音频文件本身格式不支持”问题。

原因：
- 同一首文件此前可播放、当前最终也可播放，文件可读可解码。
- 失败点在“输出设备 line 打开”阶段（设备输出层），不是文件解码层。
- 前端 JSON 解析报错是附加问题，不是主根因。

统一理解：
- **主根因**：选定输出设备在当时打开 line 失败（设备状态/参数接受度问题）。
- **附加问题**：失败响应处理不规范导致前端拿到非预期响应并触发 JSON 解析错误。

---

## 3) 中间误导点
- 误导 1：把问题归因为“歌曲格式不支持”。
- 误导 2：把“设备打开失败”和“JSON 解析失败”当作两个独立 bug。

实际应当合并为单一问题链：
设备打开失败（主） -> 响应不规范（次） -> 前端解析二次报错（表现）。

---

## 4) 最终修复点
1. 统一 POST 失败返回 JSON（不再返回 HTML 错页）。
2. `/api/local/upload-and-import` 在 JSON 解析前优先处理 multipart，避免把二进制 body 当 JSON 解析。
3. 保持诊断字段口径：
   - `configuredDevice`
   - `actualOpenedDevice`
   - `lastError`
